# 量化 CTA 绩效分析系统 — 使用说明

## 1. 系统简介

**量化 CTA 绩效分析系统**（`绩效分析.html`）是一款本地运行的 Web 分析工具，用于对 CTA / 商品期货类基金的净值进行**绩效统计**与**因子暴露度归因**。

核心能力：

- 导入基金净值，计算年化收益、波动率、夏普比率、最大回撤等基础指标
- 导入市场因子日收益率，与基金收益按日期对齐
- 全样本 OLS 多元回归，估计各因子暴露度（β）及显著性
- 滚动窗口回归，观察暴露度随时间的变化
- 可视化：暴露度柱状图、净值曲线、累计归因、因子序列分析
- 导出 CSV 格式的因子暴露度分析报告

> **运行方式**：用 Chrome / Edge 等浏览器直接打开 `绩效分析.html` 即可，无需安装后端或联网（Chart.js 需联网加载 CDN；若离线使用，可将 Chart.js 改为本地引用）。

## 3. 快速上手（5 步）

### 步骤 1：打开工具

双击或在浏览器中打开 `绩效分析.html`。首次进入会自动加载**内置模拟因子**和**示例基金净值**，并运行一次演示分析。

### 步骤 2：准备并上传基金净值

点击「上传基金净值 CSV」，选择符合格式要求的文件（见第 4.1 节）。

- 也可点击「**加载示例**」快速体验
- 「**清除数据**」可重置基金净值，保留当前因子设置

### 步骤 3：准备并上传因子收益率（可选）

- 默认使用 **6 个内置模拟因子**（趋势、动量、波动率、价值、利差、商品 Beta）
- 若有真实因子收益序列，点击「上传因子收益率 CSV」（见第 4.2 节）
- 「**恢复内置因子**」可切回演示数据

### 步骤 4：配置分析参数

1. **滚动窗口**：默认 60 个交易日，范围 15–200，用于滚动暴露度估计
2. **选择因子**：至少勾选 **2 个**因子（多元回归需要）
3. 点击「**运行因子暴露度分析**」

### 步骤 5：查看结果并导出

分析完成后，右侧面板展示摘要、回归表与图表。点击「**导出分析报告**」下载 CSV 报告。

## 4. 数据格式要求

### 4.1 基金净值 CSV

**推荐格式（两列）：**

```csv
date,nav
2024-01-02,1.0000
2024-01-03,1.0012
2024-01-04,0.9998
```

| 列 | 说明 |
|----|------|
| 第 1 列 | 日期，支持 `YYYY-MM-DD`、`YYYY/M/D`、`MM/DD/YYYY` |
| 第 2 列 | 基金单位净值（数值） |

**简化格式（仅净值列）：**

```csv
nav
1.0000
1.0012
0.9998
```

无日期时系统自动生成 `Day_1`、`Day_2`… 作为序号（不推荐用于与因子对齐）。

**要求：**

- 至少 **10 个**有效净值点
- 首行可为表头（系统自动识别）
- 分隔符支持逗号、分号、制表符
- 日收益率由相邻净值计算：`r_t = (NAV_t - NAV_{t-1}) / NAV_{t-1}`

### 4.2 因子收益率 CSV

**标准格式：**

```csv
date,mom_5d_cs,carry_cs,vol_cs,basis_cs
2024-01-02,0.0012,-0.0008,0.0021,0.0005
2024-01-03,-0.0003,0.0015,0.0009,-0.0011
2024-01-04,0.0007,0.0002,-0.0018,0.0003
```

| 列 | 说明 |
|----|------|
| 第 1 列 | 日期（格式同基金 CSV） |
| 第 2 列起 | 各因子**日收益率**（小数形式，如 0.001 = 0.1%） |

**要求：**

- 必须有表头
- 至少 **20 个交易日**有效数据
- 缺失值会被当作 0 处理
- 日期需与基金净值有**足够重叠**，对齐后样本不少于 20 天

### 4.3 与因子回测 Notebook 的衔接

若使用 `factor_construct.ipynb` 做截面因子回测，Notebook 中 `BACKTEST_RESULTS[factor_name]['daily_ls']` 即为各因子的**多空组合日收益**，可导出为本工具所需的因子列：

```python
# 示例：将回测多空日收益导出为因子 CSV
import pandas as pd

rows = []
for name, res in BACKTEST_RESULTS.items():
    if res is None:
        continue
    s = res['daily_ls'].rename(name)
    rows.append(s)

factor_df = pd.concat(rows, axis=1)
factor_df.index.name = 'date'
factor_df = factor_df.reset_index()
factor_df.to_csv('因子收益率.csv', index=False, encoding='utf-8-sig')
```

注意：

- 本工具需要的是**日收益率**，不是因子原始值或净值
- 基金净值 CSV 与因子 CSV 的**交易日历需对齐**

## 6. 输出指标与图表解读

### 6.1 基金数据概要（左栏）

| 指标 | 计算方式 | 解读 |
|------|----------|------|
| 年化收益率 | 日收益均值 × 252 | 样本期内基金年化表现 |
| 年化波动率 | 日收益标准差 × √252 | 风险水平 |
| 夏普比率 | 年化收益 / 年化波动 | 未扣除无风险利率的简单夏普 |
| 最大回撤 | 净值峰值到谷底的最大跌幅 | 极端下行风险 |
| 最终净值 | 样本期末净值 | 累计表现 |

### 6.2 分析摘要（右栏）

| 指标 | 含义 |
|------|------|
| R² | 因子能解释基金日收益变动的比例 |
| 调整 R² | 扣除因子数量后的拟合优度 |
| 年化 Alpha | 截距项 × 252，因子无法解释的超额收益 |
| 样本量 / 因子数量 | 回归使用的交易日数与自变量个数 |

### 6.3 因子暴露度回归表

回归模型：

```
R_fund,t = α + β₁·F₁,t + β₂·F₂,t + … + βₖ·Fₖ,t + ε_t
```

| 列 | 含义 |
|----|------|
| 暴露度 (β) | 因子收益每变动 1 单位，基金收益的平均变动 |
| 标准误 | 系数估计的不确定性 |
| t 统计量 | 系数 / 标准误 |
| p 值 | 统计显著性（越小越显著） |
| 显著性 | ★★★ p<1%，★★ p<5%，★ p<10% |
| VIF | 方差膨胀因子，>10 提示多重共线性风险 |

**Alpha（截距）**：控制所选因子后，基金仍具有的日均超额收益；年化 Alpha = α × 252。

### 6.4 可视化图表

| 图表 | 说明 |
|------|------|
| 因子暴露度柱状图 | 全样本 β 系数，绿色为正暴露、红色为负暴露 |
| 基金净值曲线 | 上传净值的时序走势 |
| 滚动暴露度 | 选定因子在滚动窗口内的 β 变化及 95% 置信区间 |
| 累计因子收益贡献 | 各因子 β×收益 的累计贡献 vs 拟合总收益 vs 实际收益 |
| 因子累计净值 | 对齐区间内各因子多空组合的累计表现 |
| 因子日收益 / 滚动波动率 | 单因子日收益序列与滚动年化波动率 |

切换「滚动暴露度」和「因子序列」下方的**因子标签**，可查看不同因子的细节。

## 7. 分析设置说明

### 滚动窗口

- 默认值：**60 个交易日**
- 可调范围：15–200
- 窗口越大，暴露度估计越稳定，但对结构变化反应越慢
- 建议：短周期策略用 40–60 天，长周期策略用 90–120 天

### 因子选择

- 至少选择 **2 个**因子
- 因子之间相关性过高时，关注 VIF 列
- 可先选风格差异大的因子（如动量 + _carry + 波动率），再逐步扩展

## 9. 常见问题

### Q1：提示「对齐后仅 N 个数据点，不足 20 个」

**原因**：基金净值与因子数据的**日期交集太少**。

**处理**：

- 检查两边日期格式是否一致（建议统一为 `YYYY-MM-DD`）
- 确认因子 CSV 第一列是日期而非空列
- 扩大样本区间，保证至少 20 个共同交易日

### Q2：提示「请至少选择 2 个因子」

多元回归至少需要 2 个自变量因子。勾选 2 个及以上因子后重新运行。

### Q3：R² 很低是否正常？

CTA 基金收益受交易执行、杠杆、品种选择、费用等影响，因子无法 100% 解释。R² 在 20%–60% 区间较常见；更应关注**显著暴露的因子方向**是否与策略逻辑一致。

### Q4：Alpha 显著为正/负说明什么？

在控制所选因子后，Alpha 反映**未被因子解释的系统性超额或落后**。需结合样本长度、因子覆盖是否完整来解读，不宜单独作为选基依据。

### Q5：VIF 很高怎么办？

说明因子间共线性强，β 估计不稳定。可尝试：

- 减少高度相关的因子（如只保留动量或趋势之一）
- 延长样本区间
- 对因子做正交化处理后再导入

### Q6：与 Notebook 截面回测的关系？

| 模块 | 作用 |
|------|------|
| `factor_construct.ipynb` | 计算因子、截面分组回测，输出单因子多空收益 |
| `绩效分析.html` | 将基金净值与多个因子收益回归，做**组合层面归因** |

两者互补：Notebook 回答「因子本身是否有效」，绩效分析回答「基金收益来自哪些因子暴露」。

## 10. 注意事项

1. **本地运行，数据不上传**：所有计算在浏览器内完成，适合敏感净值数据。
2. **简单 OLS 假设**：未考虑异方差、自相关校正；结论以**方向与量级**为主，精确 p 值供参考。
3. **日频对齐**：系统按**精确日期**匹配，非交易日不会自动填充。
4. **无风险利率**：当前夏普与 Alpha 年化未扣除无风险利率（与 Notebook 中 `RISK_FREE_RATE = 2%` 的设置不同，跨模块对比时需注意）。
5. **演示数据**：内置因子与示例基金为模拟数据，不可用于实盘决策。
